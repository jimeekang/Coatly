[0] 배경 / 문제 정의
================================================================================

현재 New Quote 화면에는 4개의 pricing strategy가 있음:
  - hybrid       (= Detailed Estimate, sqm 기반 정밀 견적)
  - day_rate     (= Labour × days)
  - room_rate    (= Room flat rate, 방별 단순 합산)
  - manual       (= Direct input)

문제점:
  1. "Detailed Estimate"와 "Room Flat Rate"의 경계가 모호함
     → painter 입장에서 어느 걸 써야 할지 헷갈림
  2. Room Flat Rate에는 wall/ceiling/trim 분리가 없고, condition 보정도 없음
  3. Price Rates에 detailed_estimate_anchors와 room_rate_presets가 별도 저장됨
     → 같은 정보를 두 번 관리해야 함
  4. Master Bedroom + ensuite 조합 등 변형 케이스가 없음
  5. trim 종류에 따라 가격 차이가 큰데 반영 못 함

목표:
  - Detailed Estimate와 Room Flat Rate를 "Detailed Estimate" 한 옵션으로 통합
  - 통합된 옵션 안에서 Quick / Advanced 모드 분기
  - Quick mode는 새로 설계 (방 × 사이즈 × wall/ceiling/trim 매트릭스 + multipliers)
  - Advanced mode는 기존 Detailed Estimate(sqm 기반) 그대로 유지
  - 모든 가격/% multiplier는 Price Rates에서 painter가 직접 입력 (시스템 하드코딩 금지)

4-1. utils/calculations.ts에 calculateQuickEstimate 추가
--------------------------------------------------------------------------------

export function calculateQuickEstimate(
  inputs: QuickEstimateInputs,
  rateSettings: UserRateSettings
): QuickEstimateResult;

QuickEstimateResult = {
  subtotal_cents,
  gst_cents,
  total_cents,
  labor_cents,
  material_cents,
  rooms: Array<{
    room_id, label, size, selected_surfaces, notes,
    walls_cents, ceiling_cents, trim_cents,         // base (multiplier 전)
    base_subtotal_cents,
    coating_multiplier_pct, condition_multiplier_pct,
    total_cents
  }>
}

계산 단계 (각 방):
  1) rateSettings.quick_estimate.rooms 에서 room_id 매칭
  2) matched.sizes[size] → walls_cents / ceiling_cents / trim_cents
  3) selected_surfaces에 포함된 surface만 합산:
       base_subtotal = (walls? walls_cents : 0)
                     + (ceiling? ceiling_cents : 0)
                     + (trim? trim_cents : 0)
  4) coating_pct, condition_pct 가져오기
  5) room_total = round(base_subtotal × (coating_pct/100) × (condition_pct/100))

집계:
  subtotal_cents = sum(room_total)
  labor_cents    = round(subtotal × labour_share)   ← rateSettings.pricing 재사용
  material_cents = subtotal - labor_cents
  gst_cents      = round(subtotal × gst_rate)       ← rateSettings.pricing 재사용 (호주 10%)
  total_cents    = subtotal + gst_cents

→ labour_share / gst_rate는 기존 hybrid에서 쓰는 동일 출처 사용

function hydrateQuickEstimate(rates: UserRateSettings): UserRateSettings {
  if (rates.quick_estimate) return rates;

  // 1) 기존 room_rate_presets 활용 (있으면)
  const seedFromPresets = rates.room_rate_presets.map((preset, i) => ({
    id: crypto.randomUUID(),
    label: preset.title,
    enabled_surfaces: ['walls','ceiling','trim'],
    sizes: {
      small:  splitToSurfaces(round(preset.rate_cents * 0.80)),
      medium: splitToSurfaces(preset.rate_cents),
      large:  splitToSurfaces(round(preset.rate_cents * 1.25)),
    },
    sort_order: i,
  }));

  // 2) presets 없으면 기본 템플릿 10개 (가격 0)
  const seed = seedFromPresets.length > 0 ? seedFromPresets : DEFAULT_QUICK_ROOM_TEMPLATES;

  return {
    ...rates,
    quick_estimate: {
      rooms: seed,
      coating_multipliers: {
        one_coat_refresh_pct: 70,
        two_coats_repaint_pct: 100,
        three_coats_new_plaster_pct: 140,
      },
      condition_multipliers: {
        good_pct: 90,
        average_pct: 100,
        poor_pct: 130,
      },
    },
  };
}

// industry rule of thumb: walls 50% / ceiling 25% / trim 25%
function splitToSurfaces(total_cents: number) {
  return {
    walls_cents:   round(total_cents * 0.50),
    ceiling_cents: round(total_cents * 0.25),
    trim_cents:    round(total_cents * 0.25),
  };
}
